# 出口成章｜开发路线图与验收标准

## 总目标
把“出口成章”推进到：
1. 有可运行的 iPhone App
2. 有完整 Demo 流程
3. 关键能力可实际验证
4. 通过一轮自测验收

## 里程碑

### M0：概念收敛（已基本完成）
- 项目定位
- PRD
- Demo 脚本
- MVP 范围
- API 草案

### M1：可演示原型
目标：不接真实后端，也能在 iPhone App 里完整走一遍故事。

完成标准：
- 首页可进入录音页
- 录音页可生成 Mock 转写结果
- 结果页可展示原文 / 优化稿 / 问题分析
- 训练页可展示 2~3 轮训练结果
- 灵感库页可看到归档结果
- 报告页可展示每日统计

### M2：可运行 MVP
目标：接入真实能力，形成最小可用产品。

完成标准：
- 可录真实音频
- 可调用 STT 得到真实转写
- 可调用优化接口得到真实优化稿
- 可进行训练评分
- 可归档到灵感库
- 可保存历史记录

### M3：Demo 完整版
目标：适合展示、录屏、讲解。

完成标准：
- 有 1 条稳定的演示脚本
- 有 1 套稳定的 Demo 数据
- 有 1 个可录屏的完整流程
- 价值点讲述顺畅

### M4：验收通过版
目标：不是只有页面，而是能过一轮完整自测。

完成标准：
- 主流程无明显崩溃
- 关键页面能打开
- 关键按钮可用
- Demo 可在 5 分钟内讲完
- 至少完成 1 次真实音频 -> 转写 -> 优化 -> 训练 -> 归档

## 核心主流程
1. 用户打开 App
2. 点击开始录音
3. 录入一段真实表达
4. 得到原文与优化稿
5. 查看表达问题分析
6. 进行第 2 轮复述训练
7. 查看提升结果
8. 一键归档到灵感库
9. 在报告页看到统计结果
