# 出口成章｜服务选型建议（可直接拍板版）

## 结论先行

如果现在就要追求**稳定落地、用户能用、能闭环**，我建议直接这么定：

### STT（语音转文字）
- **国内优先主选**：讯飞开放平台
- **国内备选 / 聚合型高性价比**：SiliconFlow（若其语音能力可用）/ 火山引擎语音服务
- **海外 / 实时优先**：Deepgram
- **本地 free 线**：faster-whisper

### TTS（文字转语音）
- **中文主选**：阿里云 Qwen3-TTS
- **中文稳妥备选**：讯飞 TTS
- **展示效果最强**：ElevenLabs
- **基础多语言稳妥**：Google Cloud TTS / Amazon Polly

### LLM（表达优化 / 训练反馈）
- **国内主选**：SiliconFlow + 智谱 GLM + 火山引擎豆包
- **海外聚合主选**：OpenRouter
- **后期统一网关**：LiteLLM（但先别急着上）

---

## 一、我建议直接拍板的方案

### 方案 A：最快落地，优先让产品闭环跑起来
- STT：**讯飞**
- TTS：**阿里云 Qwen3-TTS**
- LLM：**SiliconFlow + OpenRouter 双 profile**

适合原因：
- 中文场景强
- 接口成熟
- 有免费额度 / 新人额度 / 低成本试错空间
- 你现在这个项目已经有 OpenAI-compatible 多 profile 快切底座，很适合直接接 SiliconFlow / OpenRouter

### 方案 B：低成本 + 可控 + 双线并行
- STT：**本地 faster-whisper** + **HTTP fallback（讯飞 / Deepgram）**
- TTS：**阿里云 Qwen3-TTS**
- LLM：**SiliconFlow**（国内主力）+ **OpenRouter**（海外补位）

适合原因：
- 本地 free 线保留
- 网络或模型下载失败时不至于全盘卡死
- 更符合“先救活，再做强”的路线

---

## 二、具体服务名称、免费额度 / 优惠方向、我的判断

## STT：具体服务比较

### 1. 讯飞开放平台
**我建议：国内 STT 第一优先考虑它。**

搜索摘要给到的关键信息：
- 讯飞听见免费版：**每天约 1 小时录音转写额度**
- 实时语音转写：有免费试用
- 中文、方言、实时语音场景强

适合：
- 中文表达产品
- 会议 / 口述 / 灵感记录
- 需要尽快稳定上线

判断：
- **能用、稳、中文强**
- 如果你问“真要先接一家 STT，我选谁？”——**我会先选讯飞**

### 2. Deepgram
**我建议：海外实时 STT 第一梯队。**

搜索摘要给到的关键信息：
- 实时延迟低于 300ms
- 实时语音代理场景强
- 企业级功能成熟
- 免费层摘要里没给明确分钟数，但它经常有试用额度

适合：
- 海外链路
- 实时语音交互
- 英文或多语种更重的场景

判断：
- 如果你要“速度快、实时强”，它很值得接
- 但对当前“出口成章”中文优先产品，**优先级仍低于讯飞**

### 3. AssemblyAI
搜索摘要给到的关键信息：
- 多语言支持较多
- 有较成熟语音理解能力
- 免费额度摘要中没拿到明确分钟数

判断：
- 强，但不是当前中文表达 MVP 的第一选择

### 4. Google Speech-to-Text
判断：
- 多语言和企业级能力强
- 但当前你的环境、支付和网络现实下，不是最优首接项

### 5. faster-whisper
**我建议：保留为本地 free 线。**

现状：
- 包已装上
- 代码链路已接通
- 当前卡点是模型下载网络，而不是代码本身

判断：
- **应该保留，不该放弃**
- 但它不应该成为唯一生产路径

## TTS：具体服务比较

### 1. 阿里云 Qwen3-TTS
**我建议：中文 TTS 主选。**

搜索摘要给到的关键信息：
- 低延迟表现强
- 音色克隆强
- 中文场景更实用
- 阿里云模型和 API 生态较全

判断：
- 如果你要先接一个中文 TTS，**我会先接它**

### 2. 讯飞 TTS
**我建议：中文 TTS 稳妥备选。**

搜索摘要给到的关键信息：
- 每日免费调用 500 次
- 中文语音能力成熟

判断：
- 很稳，很适合中文
- 如果阿里云那边不顺，讯飞是很好的第二主线

### 3. ElevenLabs
**我建议：演示效果第一名。**

搜索摘要给到的关键信息：
- 语音自然度极强
- 适合展示与内容创作
- 免费额度摘要里没拿到特别明确数字，但有免费 / 试用策略

判断：
- 如果你要做 Demo，想让人“听完就觉得牛”，它很值得上
- 但作为中文主链路，不一定是第一选择

### 4. Google Cloud TTS / Amazon Polly
判断：
- 稳、企业级、多语言好
- 适合后续全球化或更正式云架构
- 当前不是最优先接入项

## LLM：具体服务比较

### 1. SiliconFlow（硅基流动）
**我建议：当前国内 LLM 第一主选。**

搜索摘要给到的关键信息：
- 注册即送 **2000万 Tokens**
- 完成任务可额外领 **1000万 Tokens**
- 新人认证还有 **16 元代金券**
- 小模型（7B 以下）很多免费
- 接口：`https://api.siliconflow.cn/v1`

判断：
- 对你现在这个项目，**几乎就是最合适的国内第一接入对象**
- 原因：
  - 免费额度大
  - OpenAI-compatible
  - DeepSeek / Qwen 都能试
  - 国内直连更现实

### 2. OpenRouter
**我建议：海外多模型对比的第一主选。**

搜索摘要给到的关键信息：
- 免费模型以 `:free` 结尾
- 免费用户：**每日 50 次请求 / 20 RPM**
- 充值 ≥10 credits 后，免费模型日上限可到 **1000 次**
- 支持 300+ 模型

判断：
- 很适合你现在的多模型试错
- 我建议它作为**第二主 profile**

### 3. 智谱 GLM
搜索摘要给到的关键信息：
- **2000万 Token 永久有效**
- `GLM-4-Flash` 永久免费无上限

判断：
- 非常值得接
- 如果你问“国内除了 SiliconFlow，还该接谁？”——**智谱一定值得列入**

### 4. 火山引擎 / 豆包 / 方舟
搜索摘要给到的关键信息：
- **每日 200 万 Token**
- 新用户额外 50 万 Token

判断：
- 很适合自动化脚本、规律性业务
- 值得作为第三候选

### 5. 阿里云百炼 / 百度千帆
判断：
- 都值得保留在候选池
- 但如果今天就拍板，我会优先：
  - SiliconFlow
  - 智谱
  - 火山引擎
  - OpenRouter

## 四、为什么我这样选

因为你现在要的不是“理论上最强”，而是：

- 能真的接上
- 成本别离谱
- 免费 / 优惠能先撑起 MVP
- 中文效果要行
- 稳定能闭环
- 用户真能用上

按这个标准，我现在的拍板顺序就是：

### STT
**讯飞 > Deepgram > faster-whisper（本地线）**

### TTS
**阿里云 Qwen3-TTS > 讯飞 > ElevenLabs**

### LLM
**SiliconFlow + OpenRouter 双主线**
再补：**智谱 GLM / 火山引擎**

## 五、下一步工程动作

我建议马上这么落地：

1. 把 **SiliconFlow** 配进 `llm-profiles.json`
2. 把 **OpenRouter** 也配进去，形成双 profile
3. STT 侧开始对接 **讯飞** 的 HTTP provider
4. 保留 **faster-whisper** 本地线，继续解决离线模型问题
5. TTS 侧开始预留 **阿里云 Qwen3-TTS** provider 插槽

这样就是你要的：

**双线并行、稳定落地、用户真能用、闭环真能跑。**
