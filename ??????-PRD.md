# 出口成章 PRD（MVP v1）

## 1. 产品定义

**产品名：** 出口成章  
**产品形态：** iPhone App（首版）  
**一句话定位：** 把随口一说，变成清晰表达；把一个点子，推进成可验证产品。  

出口成章不是单纯的语音转文字工具，而是一个“表达增强 + 训练 + 灵感归档”的个人表达操作系统。首版先聚焦最小闭环：录音、转写、优化、训练、归档、回看。

## 2. 目标用户

1. 职场用户：开会发言、汇报、复盘
2. 创作者：口播、选题、灵感整理
3. 创业者 / 产品经理：把点子说清楚、整理清楚、沉淀成产品资产
4. 求职者 / 学生：面试、自我介绍、演讲训练

## 3. 用户核心痛点

1. 想法有，但表达不顺
2. 现有 STT 只会记，不会帮用户说得更好
3. 灵感零散，后续难归档、难复用
4. 很难低成本反复练习表达

## 4. MVP 核心目标

MVP 只验证三件事：

1. 用户是否会愿意把口头表达丢进来
2. 用户是否会对“原文 vs 优化稿对照”有明显价值感知
3. 用户是否会愿意做第二遍复述训练

## 5. 首版功能范围

### 5.1 首页
- 今日入口：开始录音 / 导入语音
- 最近 3 条内容
- 今日表达统计卡片
- 快速进入：灵感库 / 报告 / 训练

### 5.2 录音转写页
- 长按录音 / 点击开始
- 录音中计时
- 结束后进入转写状态
- 记录场景：汇报 / 口播 / 灵感 / 面试 / 通用

### 5.3 结果页（核心页）
- 左栏：原文
- 右栏：优化稿
- 问题分析：口头禅、重复、逻辑跳跃、句子过长
- 改写模式：简洁 / 正式 / 口播 / 汇报
- CTA：开始训练 / 归档灵感

### 5.4 训练页
- 展示标准稿
- 用户进行第二轮复述
- 对比前后改进情况
- 最多三轮
- 给出“出口成章率 / 清晰度 / 完整度”分数

### 5.5 灵感库页
- 自动归类内容：产品点子 / 内容选题 / 功能需求 / 商业模式 / 待验证假设
- 每条卡片显示：标题、标签、整理稿、下一步动作
- 支持按标签筛选

### 5.6 报告页
- 今日表达字数
- 训练次数
- 优化次数
- 高频问题
- 最佳一句

## 6. 首版不做

- 社交关系和分享体系
- 复杂的团队协作
- 真正的知识图谱引擎
- 多角色权限系统
- 过重的增长玩法

## 8. 成功标准

### 产品侧
- 用户 1 分钟内能完成一次录音并看到优化结果
- 用户能明确看见原文与优化稿差异
- 用户愿意开始至少一轮训练
- 用户愿意把结果归档进灵感库

### Demo 侧
- 5 分钟内可完整演示一遍流程
- 每个环节都有清晰价值展示
- 不依赖真实后端也能演示完整故事

## 9. 首版技术建议

### 前端
- SwiftUI
- 本地 Mock 数据先跑通 UI 与流程

### 后端（后接）
- 语音转写：先接高质量 STT
- 文本优化：LLM 改写 + 问题分析
- 归档：结构化入库
- 报告：按日聚合

## 10. 版本推进顺序

### V0：Demo 版
- 纯前端 + Mock 数据
- 演示完整流程

### V1：可用 MVP
- 接入真实转写
- 接入真实改写
- 保存本地历史

### V2：增强版
- 灵感网络
- 报告体系增强
- 产品孵化链路
