# 出口成章 Demo 演示脚本

## Demo 目标
在 3~5 分钟内让人看懂三件事：
1. 这不是普通 STT
2. 它能帮用户把表达变好
3. 它能把一个灵感沉淀成后续产品资产

## 演示主线

### Step 1：进入首页
话术：
“这是出口成章，一个帮你把随口表达变成清晰内容的 iPhone App。首页就给你三个入口：开始录音、继续训练、回看灵感库。”

展示点：
- 今日统计卡片
- 最近记录
- 录音主按钮

### Step 2：开始录音
演示语料建议：
“我想做一个结合语音转文字和表达训练的产品，它不只是把你说的话记下来，还会帮你整理成更有逻辑的版本，并且让你重新说一遍，看看有没有进步。”

话术：
“用户只要自然说一段话，系统会先完成高质量转写。”

展示点：
- 录音中状态
- 场景选择：灵感 / 汇报 / 口播 / 面试

### Step 3：结果页
话术：
“这里是出口成章最核心的页面。左边是原文，右边是优化稿。用户不止能看到最后结果，还能清楚知道自己哪些地方说乱了、重复了、啰嗦了。”

展示点：
- 原文 vs 优化稿
- 口头禅统计
- 逻辑结构建议
- 改写模式切换

### Step 4：训练模式
话术：
“接下来不是结束，而是训练开始。系统会给你一个标准稿，你可以照着再说一遍。说完后，它会比较你前后两次表达的差异，告诉你清晰度和结构性有没有提升。”

展示点：
- 标准稿
- 第二轮录音
- 提升结果
- 最多三轮训练

### Step 5：归档到灵感库
话术：
“如果这段内容是一个点子，而不是一次普通表达，它可以一键进入灵感库。系统会自动识别这是产品点子、内容选题还是待验证假设，并生成标题、标签和下一步动作。”

展示点：
- 自动分类
- 标签
- 下一步动作
- 状态管理

### Step 6：报告页
话术：
“每天结束后，用户可以看到自己今天说了多少、练了多少、哪些问题最常见，以及自己表达能力的变化趋势。”

展示点：
- 今日字数
- 训练次数
- 高频问题
- 最佳一句

## Demo 成功关键
1. 一定突出双栏对照
2. 一定突出训练闭环
3. 一定突出灵感归档后的产品化价值
4. 不要讲太多技术，先讲用户收益

## Demo 结束总结话术
“出口成章不是另一个语音转文字工具，而是一个把表达、训练和灵感沉淀连起来的产品。它帮助用户把脑中的想法说出来、理清楚、练熟练，并最终变成可执行的内容和产品资产。”
