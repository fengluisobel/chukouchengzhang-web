# 出口成章｜Web端可落地闭环Demo开发计划

> 目标：基于现有Web原型，接入真实AI能力，打造一个可在手机浏览器中完整闭环使用的Demo，并为后续微信小程序迁移做好准备。

---

## 一、现状分析

### 已有基础（无需从零开始）
| 模块 | 现状 | 质量 |
|------|------|------|
| 前端6页面 | 首页/录音/结果/训练/灵感库/报告 结构完整 | ⭐⭐⭐ 可用 |
| 后端API | Node.js HTTP server，RESTful API 完整 | ⭐⭐⭐ 可用 |
| 数据存储 | JSON文件持久化，导入导出支持 | ⭐⭐ 够用 |
| Provider抽象 | local/http 双模式，易扩展 | ⭐⭐⭐ 优秀 |
| LLM切换 | 多profile配置体系已有 | ⭐⭐⭐ 优秀 |
| 浏览器STT | Web Speech API 已接入 | ⭐⭐ 基础可用 |
| 浏览器TTS | SpeechSynthesis 已接入 | ⭐⭐ 基础可用 |

### 核心缺口（阻碍闭环落地）
| 缺口 | 影响 | 优先级 |
|------|------|--------|
| LLM未接入真实模型 | 优化稿是模板拼接，无真实AI价值 | **P0** |
| STT未接入真实服务 | 浏览器API识别率低，仅支持Chrome | **P0** |
| 移动端体验粗糙 | 侧边栏布局不适合手机，无PWA | **P1** |
| 数据仅本地JSON | 换设备丢失，无法多终端同步 | **P2** |

## 二、目标定义

### 2.1 核心目标
**让「出口成章」在移动浏览器中完成真实闭环：**

用户打开手机 → 长按录音 → 讯飞STT转写 → LLM生成优化稿 → 查看问题分析 → 复述训练 → AI评分 → 归档灵感 → 查看日报

### 2.2 成功标准
- [ ] 用真实语音录入，10秒内得到转写结果
- [ ] 优化稿由真实LLM生成，非模板拼接
- [ ] 训练评分有真实参考价值
- [ ] 手机浏览器体验流畅（不用缩放、拖动）
- [ ] 断网或刷新后数据不丢失
- [ ] 5分钟内可完整Demo一遍流程

### 2.3 微信小程序预埋目标
- 后端API设计兼容小程序鉴权（JWT/token）
- 录音逻辑抽象为可替换层（浏览器API ↔ 小程序录音API）
- 文件存储抽象为可替换层（本地JSON ↔ 云存储）

## 三、技术选型

### 3.1 后端（升级而非重写）
- **框架**：保留Node.js原生HTTP，轻量够用
- **LLM**：SiliconFlow（主）+ OpenRouter（备）
  - 原因：OpenAI-compatible，已有llm-switch.js底座，免费额度大
- **STT**：讯飞开放平台（主）+ 浏览器Web Speech（备）
  - 原因：中文强、有免费额度、HTTP接口易接入
- **TTS**：阿里云Qwen3-TTS（预留接口，暂不接入）
- **数据**：保留JSON文件存储（Demo阶段够用）

### 3.2 前端（改造而非重写）
- **框架**：保留原生HTML/CSS/JS（为小程序迁移保留灵活性）
- **样式**：重写为移动端优先的响应式布局
- **交互**：添加页面转场动画、录音动效、加载状态
- **PWA**：添加manifest和service worker，支持添加到主屏

### 3.3 为什么不选Vue/React/uni-app？
- 当前阶段最重要的是**验证AI能力闭环**，而非前端框架
- 原生JS更容易迁移到微信小程序（小程序语法接近原生JS）
- 减少构建依赖，降低Demo阶段维护成本
- 如果后续确认要长期维护，再迁移到uni-app不迟

---

### Phase 2：真实LLM接入（Day 1-2）
**目标：让优化稿和训练反馈由真实AI生成**

- [ ] 2.1 接入SiliconFlow profile
  - 配置文件：`app/config/llm-profiles.json`
  - 模型：DeepSeek-V3 / Qwen2.5
- [ ] 2.2 实现真实`polish`接口
  - 提示词工程：把rawText → polishedText + issues + suggestedTags
  - 支持scene（汇报/口播/面试/灵感）和mode（简洁/正式/口语/高管）
- [ ] 2.3 实现真实`training/evaluate`接口
  - 提示词工程：对比polishedText和attemptText，给出结构/清晰/润色三维评分
- [ ] 2.4 添加LLM失败fallback
  - 网络超时 → 本地启发式规则兜底
  - 返回标识让用户知道"当前是AI优化/本地优化"

**提示词草案**（参考`API与提示词草案.md`）：
```
【表达优化提示词】
你是一位表达教练。用户说了一段话，请你：
1. 生成优化稿（保留原意，提升结构感和清晰度）
2. 列出3-5个表达问题（口头禅、逻辑跳跃、句子过长等）
3. 给出2-3个标签分类
4. 给出下一步行动建议

场景：{scene}
风格：{mode}
原文：{rawText}
```

**交付物**：优化稿由LLM生成，训练评分有真实差异

- [ ] 3.3 双STT策略

  - 备用讯飞HTTP接口（准确率高、支持全浏览器）
  - 用户可手动选择

**交付物**：手机浏览器可录音，并得到准确转写
